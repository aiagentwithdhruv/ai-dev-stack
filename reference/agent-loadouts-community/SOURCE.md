# Reference: Agent Loadouts (community)

Origin: distilled from a public "agent loadouts" template repository — a vendor-neutral
system for giving AI tools persistent project context (loadouts, skills, agents, memory).

This folder keeps only the parts that are **generic, reusable, and not already covered**
by the flagship's `docs/AGENTS.md`, `docs/SKILLS.md`, and `docs/LOADOUT.md`.

## What's here

| File | Why it's kept |
|------|---------------|
| `memory-vs-context.md` | The flagship has agent/skill/loadout docs but **no memory-system guide**. This covers persistent recall across sessions: memory vs context, what to save, entry formats, cleanup. |
| `agent-vs-skill-decision.md` | The decision matrix (skill vs agent), orchestrator routing, and composition chains — complements the flagship's terser `docs/AGENTS.md` / `docs/SKILLS.md`. |
| `role-playbooks.md` | Six role-based "starter loadouts" (freelancer, sales, content, developer, agency, student). The flagship has none; these are a useful onboarding/sales layer. |

## What was intentionally excluded (not folded)

- All author/brand identifiers (personal name, GitHub handle, LinkedIn links, badges).
- A product-specific example loadout tied to a named API course/product — employer/
  product-specific, not generic.
- Links to a private "automation" repo and its proprietary skill catalog.
- Marketing copy, screenshots, and the README's promotional framing.
- Specific commercial tool names were kept only where they read as neutral examples;
  any reference implying a specific private system was genericized.

Everything below was rewritten to be employer/vendor-neutral. Example names, domains,
and numbers are illustrative placeholders.
